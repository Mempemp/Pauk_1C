# Бэклог фабрики (вне `pauk-product/`)

Идеи и долг, которые **не** входят в копируемый пакет. Реализация по приоритету.

| ID | Тема | Суть | Статус |
|----|------|------|--------|
| BL-001 | MCP для БСП | Версий БСП и объёма кода в типовых конфигурациях много: постоянный `grep` по выгрузке тяжёл и не гарантирует соответствие **вашей** ревизии. Нужен **отдельный слой** быстрого поиска: индексация по **конкретной** поставке/версии БСП, семантический или символьный поиск, контракт «какой снимок/версия проиндексирована». Рассматривать **после** доработок по поиску в репозитории и (при необходимости) универсальных code-MCP (Serena, RLM и т.п.) — уточнить, не закрывают ли они сценарий. | open |

## BL-001 — детализация (критерий готовности, черновик)

- Можно по запросу разработчика найти **релевантные** модули/процедуры БСП **в той ветке версии**, с которой работают, без полного прогона репо вручную.
- Понятно, **какая** редакция БСП лежит в индексе; при смене поставки — явное обновление снимка.
- Тело навыка `ssl_patterns` остаётся **без** ссылок на внутреннюю кухню фабрики; про этот инструмент можно описать **в продукте** только как «при подключённом MCP …», когда MCP появится.
