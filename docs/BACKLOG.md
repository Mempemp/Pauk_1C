## BL-001 — детализация (критерий готовности, черновик)

- Можно по запросу разработчика найти **релевантные** модули/процедуры БСП **в той ветке версии**, с которой работают, без полного прогона репо вручную.
- Понятно, **какая** редакция БСП лежит в индексе; при смене поставки — явное обновление снимка.
- Тело навыка `ssl_patterns` остаётся **без** ссылок на внутреннюю кухню фабрики; про этот инструмент можно описать **в продукте** только как «при подключённом MCP …», когда MCP появится.

## BL-002 — детализация (закрыто, v1)

**Критерии, которые закрыты:**

- **Явный шаг** между формулировкой и дальнейшим выбором тел навыков для **объёмных** задач: `ARTIFACT-ROUTER` + нулевой этап в `full-pipeline` (роль `task-analyst`), с полями классификации, рекомендуемым `route` и пофазовым handoff.
- **Для обычных** задач по-прежнему действуют `SKILLS-CATALOG` (сигналы → `skill_id`) и прогрессивное чтение `SKILL.md`; **отдельного** файла-команды не требуется, артефакт `ARTIFACT-ROUTER` не обязателен.
- **Правило оркестратора:** `pauk-entry` шаг 0 при использовании сценария из `.cursor/commands/` (например `flow-full-pipeline.md`); канон сценариев — `WORKFLOW.md` (пресеты).

**Остаётся вне v1 (не блокирует закрытие):** автоматизированный классификатор, embeddings/MCP-skill-registry (см. [CONCEPT.md](CONCEPT.md) §4), отдельный лёгкий `skill` только с матрицей маршрутизации — при появлении **новой** задачи в бэклоге.

## BL-006…BL-014 — порядок и зависимости (черновик)

- Рациональный порядок: **006 → 007 → 008** (правила/навыки по слоям), затем **009** (MCP), **010** (тяжёлый порт метаданных), **011** (субагент), **012–013** (агенты и оставшиеся правила), **014** (команды/SDD).
- **010** и **001/004** не блокируют друг друга полностью, но сценарии «поиск по конфигурации» должны в итоге **не противоречить** (см. `COMOL-CONVERSION-MATRIX` §7).
- Утилитарные навыки Comol (`mermaid-diagrams`, `powershell-windows`, `img-grid-analysis`) не выделены в отдельные BL: при необходимости — мини-задача после **010** или вне пакета.

## BL-004 — детализация (черновик)

- **Цель**: меньше «лестницы» из полных чтений файлов; больше точечных hit’ов с путями и выдержками.
- Минимальный шаг: подключить и настроить MCP поиска по репо (или зафиксировать, почему выбран встроенный инструмент + политика использования).
- Политика для агентов/правил: сначала поиск/индекс, затем раскрытие файла; границы scope (только `pauk-product/`, вся фабрика, исключения).
- Связь с **BL-001**: не плодить два независимых поиска без схемы; приоритет — один контракт «где ищем».

## BL-005 — детализация (черновик)

- **Объём**: то, что копируется в пакет продукта и длительно в контексте (`.cursor/rules`, `SKILL.md` в продукте, промпты субагентов, пользовательские инструкции пакета).
- **Исключения/осторожность**: внутренняя документация фабрики, обсуждения с заказчиком, юридические формулировки — не переводить слепо; список исключений.
- **Контроль качества**: единый глоссарий 1С-терминов (EN) или ссылка на внешний; вычитка носителем по критичным инструкциям.
- Критерий готовности: основной продуктовый пакет на EN; в README/релиз-ноутах указано, что EN — рабочий язык инструкций.
