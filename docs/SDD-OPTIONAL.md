# Опциональные SDD-фреймворки (Memory Bank, OpenSpec, TaskMaster, Spec Kit)

Материал вынесен из Comol `sdd-integrations.mdc` **в фабрику**: в копируемом пакете `pauk-product` **нет** длинного описания этих инструментов — иначе раздувается контекст без пользы для репозитория без `memory-bank/`, `openspec/` и т.д.

**Совместимость с Pauk:** маршрут и навыки по-прежнему задаются `pauk/routing/PROTOCOL.md` и `SKILLS-CATALOG.md`. SDD-артефакты (spec, tasks, memory-bank) — **источник контекста и задач для человека**, а не отдельный «вход» с той же тяжестью, что `pauk-entry.mdc`. Перед использованием проверьте наличие папок/файлов в **вашем** репозитории (см. снимок Comol, таблицу Detection в `sdd-integrations`).

**Когда вести в проект детальную внутреннюю инструкцию:** отдельное решение команды; сюда в фабрику смыслы не копируются, чтобы не дублировать поставку Pauk.
