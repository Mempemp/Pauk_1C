# Концепция Pauk (кратко)

Живой конспект для **разработчиков фабрики**. Не копируется в `pauk-product/`. Детали исполнения — только в поставке и в узких `docs/*`.

**Поддержка:** при смене маршрутизации, MCP, пресетов или осей навыков обновляйте этот файл вместе с правками (чеклист — **`.cursor/rules/pauk-factory.mdc`**, раздел про `CONCEPT.md`).

---

## 1. Цель

- Качество и автономность задач по 1С и смежным темам.
- **Смена модели/IDE** не ломает контракт слоёв A–D (инструкции + опциональные инструменты, без привязки к конкретному LLM API).
- Расширение через **опциональные** MCP и навыки, без ядра «под один сервер».

---

## 2. Слои (логика, не обязательно отдельные репозитории)

| Слой | Содержание | Зависимости |
|------|------------|-------------|
| **A** | Политика, контракт задачи, безопасность | — см. [LAYER-A.md](LAYER-A.md) |
| **B** | Оркестрация: подзадачи, субагенты, handoff, критерии остановки | A |
| **C** | Навыки (домен 1С: BSL, метаданные, запросы, …) | A; при необходимости B |
| **D** | Маршрутизация навыков (и опционально MCP) из формулировки задачи | A, каталог C |
| **E** | Инструменты: MCP, CLI, RAG | контракты A/D |

**Фабрика vs поставка:** мета-разработка в корне репо и `docs/`; в прикладной проект — только **`pauk-product/`** ([PRODUCTION-BUNDLE.md](PRODUCTION-BUNDLE.md)).

## 4. Оси навыков (1С)

| Ось | Содержание | Сейчас |
|-----|------------|--------|
| **1** | Платформа: запросы, модули, формы, клиент—сервер, HTTP, метаданные, макеты MXL, внешние EPF/ERF, корень выгрузки, … | `pauk-product/pauk/routing/SKILLS-CATALOG.md` (в т.ч. `mxl_layouts`, `epf_erf_external`, `configuration_structure` и ранее существовавшие `skill_id`) |
| **2** | Прикладной домен конкретной конфигурации | по мере необходимости узкие навыки; не разворачивать «всю типовую» в оснастку без повтора |
| **3** | Процессы: ревью, пайплайн, роли | в субагентах и `WORKFLOW.md`; навыки не заменяют роль оркестратора |

Принцип производства навыка (источники → отбор → сжатие → строка в каталоге) — в **`.cursor/rules/pauk-factory.mdc`** (раздел про навыки в `pauk-product/`).

---

## 5. Правила, навыки, субагенты

- **Rules:** жёсткие инварианты, малый постоянный контекст.
- **Skills:** подключаемый домен по маршруту D.
- **Субагенты / workflow:** оркестрация B, условные шаги (в т.ч. крупный blueprint без правок кода до принятия). Черновик ролей и handoff — [SUBAGENTS.md](SUBAGENTS.md); порядок до первого узла — [ROUTING-ENTRY.md](ROUTING-ENTRY.md).

---

## 6. Семантика тяжёлых задач (кратко)

Класс задач вроде обмена/миграции («данные есть, семантики нет»): сначала осмысление рисков и недостающих данных, план и **набор skill_id** оси 1 — ближе к **B** (субагент, пресет, `ARTIFACT-ROUTER`), чем к навыку оси 2, пока нет стабильного прикладного повтора. Детали сценариев — в `WORKFLOW.md` и промптах субагентов.
