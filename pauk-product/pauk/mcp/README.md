# MCP

Конфигурации и описание MCP-серверов для этой оснастки в прикладном репозитории.

## Как пользоваться

Состав серверов и набор инструментов задаётся **локальной** конфигурацией проекта (например `.cursor/mcp.json` в корне репозитория). Имена и параметры инструментов нужно брать из фактически доступной в сессии схемы, а не из памяти модели.

При работе с кодом и метаданными 1С по возможности **сначала** используйте подключённые инструменты поиска, метаданных и проверки, затем точечное чтение файлов. Массовый текстовый поиск по дереву проекта — **последний** резерв, если без него задачу закрыть нельзя или специализированные средства недоступны.

## Формы и метаданные форм

Если в проекте подключены MCP для **разбора макета формы**, **XSD**, **проверки XML** или **поиска примеров форм**, зафиксируйте здесь **имена инструментов** и типовую цепочку (поиск аналога → разбор → правка `Form.xml` → проверка → компиляция/загрузка). Этапы без привязки к серверам — в `pauk/skills/form_patterns/reference/form-metadata-xml-pipeline.md`.
