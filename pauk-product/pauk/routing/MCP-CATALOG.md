# Каталог опциональных MCP

Серверы **не** входят в поставку; без них оснастка полностью работоспособна. Наличие сервера судить **только** по схеме инструментов MCP в сессии (дескрипторы клиента), не по памяти. Если сигналы ниже сработали **и** инструменты этого сервера в схеме **есть** — читать **`pauk/mcp/route-<mcp_id>.md`** и действовать по нему. Контракт шага 1 — **`PROTOCOL.md`**.

| mcp_id | Назначение |
|--------|------------|
| bsl-platform-context | Справка по встроенному языку и объектной модели из каталога установки платформы 1С |

## Сигналы → mcp_id

Подсказки по **тексту задачи**, **путям**, **вложениям**, **`@`-ссылкам**, при наличии — открытым файлам. Несколько сигналов — несколько строк; один `mcp_id` не дублировать.

| Сигнал | mcp_id |
|--------|--------|
| Вопрос или правка про **глобальные** процедуры/функции платформы, **встроенные** методы/свойства типов, **конструкторы**, **сигнатуры** встроенного API, явная отсылка к **«Синтаксис-помощнику»** / справке **платформы** (не конфигурации) | **bsl-platform-context** |
