# Каталог `Technical-Design/` (выход пресета `technical-design`)

**Назначение:** зафиксировать в репозитории **ТЗ** по эпику: согласованная формулировка, архитектура и (при необходимости) blueprint, затем **нарезка на задачи**. Реализация кода **не** входит в этот пресет — дальше человек выбирает обычный чат, `full-pipeline` или иной процесс.

## Корень каталога

- Рекомендуемое имя папки в корне прикладного репозитория: **`Technical-Design/`** (латиница, без пробелов; другое имя — только по явной договорённости с человеком).
- В корне **`Technical-Design/README.md`** — краткое описание эпика, дата/версия, **оглавление**: таблица или список ссылок на подпапки задач.

## Подпапка одной задачи

Имя: **`Txx-<краткий-slug>/`**, например `T01-import-orders/`, где `xx` — порядковый номер (два символа или больше по согласованию команды).

В **каждой** подпапке **ровно два** markdown-файла со **взаимными ссылками** в первых строках тела (после опционального YAML frontmatter):

| Файл | Содержание |
|------|------------|
| **`task.md`** | Постановка **этой** задачи: цель, входы/выходы, критерии приёмки, зависимости от других `Txx-*`, риски. Вверху ссылка на `./solution-design.md`. |
| **`solution-design.md`** | Дизайн решения **под эту задачу**: границы модулей, КС, данные, порядок шагов реализации (без полного BSL, если это не цель). Вверху ссылка на `./task.md`. |

### Минимальный шаблон взаимных ссылок

В **`task.md`** сразу после заголовка H1:

```markdown
**Дизайн:** [solution-design.md](./solution-design.md)
```

В **`solution-design.md`**:

```markdown
**Постановка:** [task.md](./task.md)
```

Далее — содержательные разделы по смыслу.

## Согласованность с чатом

- Нумерация и формулировки задач должны совпадать с тем, что **явно подтвердил** человек на шагах аналитики / архитектора / blueprint.
- Секреты, строки подключения, внутренние URL — **не** вносить; плейсхолдеры по политике **`pauk/policy/A-INVARIANTS.md`**.

## Связь с маршрутом Pauk

Пресет **`technical-design`** и порядок ролей — **`pauk/subagents/WORKFLOW.md`**, команда **`.cursor/commands/flow-technical-design.md`**. Первый заход **`task-analyst`** опирается на **`pauk/routing/ARTIFACT-ROUTER.md`** до заполнения каталога **`Technical-Design/`**.
