# Form.xml: граница с form_patterns и валидация

Индекс навыка — [../SKILL.md](../SKILL.md). MCP — [../../../mcp/README.md](../../../mcp/README.md). Конвейер «аналог → XSD → проверка → загрузка» — [../../form_patterns/reference/form-metadata-xml-pipeline.md](../../form_patterns/reference/form-metadata-xml-pipeline.md).

## 1. Где что лежит

| Тема | Навык |
|------|--------|
| Модуль формы, директивы, `&НаСервере`, события, оповещения, динамические списки | **`form_patterns`** |
| Структура **`Form.xml`**, привязка данных, команды, **проверка целостности XML формы** | этот файл + **`metadata_manage`** |

## 2. Типовые ошибки при правке Form.xml

Имеет смысл проверить перед коммитом:

| # | Проверка |
|---|-----------|
| 1 | Корень `<Form>`, версия схемы (например `2.17`) согласована с платформой проекта |
| 2 | Наличие и корректность командной панели по соглашениям конфигурации |
| 3 | Уникальность **id** элементов формы |
| 4 | Уникальность **id** реквизитов формы |
| 5 | Уникальность **id** команд |
| 6 | Наличие «сопутствующих» узлов там, где платформа ожидает парные элементы (контекстное меню, расширенные подсказки — по политике вашей выгрузки) |
| 7 | **DataPath** указывает на существующий реквизит формы или объекта |
| 8 | Кнопки со ссылкой на команду — команда объявлена |
| 9 | У обработчиков событий — непустые имена процедур в модуле формы |
| 10 | У команд — действие (handler) |
| 11 | Не более одного основного реквизита (`MainAttribute`), если правило применимо |

После существенных правок — **`get_xsd_schema`** для типа формы и **`verify_xml`** (см. README MCP).

## 3. Навигация по большому Form.xml

Для анализа без полного чтения файла используйте MCP **`inspect_form_layout`** / **`search_forms`**, если сервер доступен; иначе — точечный поиск по имени элемента в XML и модуле формы.

## 4. Типовые каркасы UI

Архетипы «документ», «список», «мастер» и именование групп — в **`form_patterns`** (области, сценарии). Здесь не дублируем длинные схемы разметки.
