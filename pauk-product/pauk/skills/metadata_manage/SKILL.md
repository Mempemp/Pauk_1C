# Метаданные и Form.xml (срез)

## Зона ответственности

Этот навык покрывает **фазу A** переноса сценария «управление метаданными» из внешнего набора правил **в урезанном виде**:

| Входит | Не входит (вне пакета / другие навыки) |
|--------|----------------------------------------|
| Структура **XML-выгрузки**, типовые объекты, краткий **JSON-DSL** для черновиков | **CF/CFE**, **БД**, **СКД**, **MXL**, **роли**, **ЕПФ**, подсистемы, интерфейс команд, справка как отдельный домен |
| **Проверка** XML объекта метаданных и **Form.xml** (чеклисты + MCP) | Запуск **`1c-meta-*` / `1c-form-*`** скриптов Comol — не поставляются; при наличии у команды — используйте их документацию и CI |
| Связка с **MCP** (`verify_xml`, `get_xsd_schema`, поиск форм/метаданных) — см. `pauk/mcp/README.md` | Живые операции с ИБ, выполнение кода в базе — не описываются здесь |

**Модуль управляемой формы**, события, клиент-сервер, `Form.Module.bsl` — навык **`form_patterns`**. Запросы и ВТ — **`query-patterns`**. БСП — **`ssl_patterns`**. Общий стиль BSL — **`coding-standards`**.

## Когда применять

- Путь или задача про **`Configuration.xml`**, **`Catalogs/`**, **`Documents/`**, **`InformationRegisters/`** и т.п. в **выгрузке** или EDT.
- Нужно **добавить/изменить** реквизит, ТЧ, измерение, форму в **XML метаданных**, не только текст в `.bsl`.
- Нужна **валидация** структуры **объекта** или **`Form.xml`** после ручной правки или генерации.
- Нужен **черновик объекта из JSON** (DSL) — см. [reference/meta-export-overview.md](reference/meta-export-overview.md); загрузка в конфигурацию — штатно через **Конфигуратор / EDT** или **ваши** скрипты.

## Как пользоваться навыком

1. **[reference/meta-export-overview.md](reference/meta-export-overview.md)** — выгрузка, типы объектов, JSON-DSL, проверки метаданных, MCP.
2. Операции **ИБ ↔ каталог** (Designer, `LoadConfigFromFiles`, `DumpConfigToFiles`, логи) — [../reference/infobase-sync.md](../reference/infobase-sync.md), не в теле этого навыка.
3. **[reference/form-xml-validate.md](reference/form-xml-validate.md)** — граница с `form_patterns`, проверки `Form.xml`, конвейер XML.
4. **[checklist-metadata-form.md](checklist-metadata-form.md)** — перед фиксацией правок в метаданных или форме.
5. При работе **с кодом в форме** — [form_patterns/SKILL.md](../form_patterns/SKILL.md); при **запросах в СКД** этого навыка недостаточно — `query-patterns` и инструменты проекта.

## Субагент и сложные сценарии

Для **многошаговых** сценариев по выгрузке (создать/править объект → **Form.xml** → валидации → исправления) в цепочке субагентов используйте роль **`metadata-manager`**: промпт **`pauk/subagents/metadata-manager.md`**, условия включения — **`pauk/subagents/WORKFLOW.md`**. Лёгкие правки можно вести без выделения этой роли — по чеклисту в этом навыке.
