# Чеклист: метаданные и Form.xml

Кратко перед фиксацией. Полные правила — в `SKILL.md` и `reference/*.md`.

## Объект метаданных (XML)

- [ ] Путь и регистрация в `Configuration.xml` согласованы с командой / типовым процессом.
- [ ] Типы ссылок (`CatalogRef.…` и т.д.) существуют в этой выгрузке.
- [ ] Проверка: MCP `get_xsd_schema` + `verify_xml` **или** загрузка в EDT/конфигуратор без ошибок.
- [ ] Перед удалением объекта: поиск вхождений (MCP / поиск по коду проекта).

## Form.xml

- [ ] Не смешивать с правками модуля: логика — в `form_patterns`, структура XML — здесь.
- [ ] Уникальность id, ссылки DataPath и команды, обработчики событий заданы.
- [ ] `verify_xml` для формы при доступном MCP.

## Общее

- [ ] Задействованы узкие навыки: при коде формы — `form_patterns`; при запросах в СКД — `query-patterns`; при БСП — `ssl_patterns`.
