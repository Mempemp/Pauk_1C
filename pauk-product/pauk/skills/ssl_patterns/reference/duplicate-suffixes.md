# Правила 8–9: не дублировать БСП; суффиксы и среда

## Правило 8: не дублируй готовую функциональность библиотеки

| Часто пишут вручную | Сначала смотри в вашей поставке БСП |
|---------------------|------------------------------------|
| получение реквизитов по ссылке | `ЗначениеРеквизитаОбъекта`, `ЗначенияРеквизитовОбъекта` |
| проверки | `ЗначениеЗаполнено` (платформа) + `ПроверитьПараметр` и аналоги в **вашей** ревизии |
| шаблоны, склонения | `СтроковыеФункции*`; см. [common-modules-strings-cs](common-modules-strings-cs.md) |
| почта | `РаботаСПочтовымиСообщениями`, если подсистема в конфигурации |
| курсы валют | `РаботаСКурсамиВалют` (имена — по метаданным) |
| журнал | [journal-files-patterns](journal-files-patterns.md) |

Двойник (свой код + похожий в БСП) даёт два места сопровождения и риск рассинхрона при обновлении.

## Правило 9: суффикс модуля и среда

| Суффикс | Среда / назначение |
|---------|--------------------|
| без суффикса | серверный общий модуль |
| `Клиент` | только клиент |
| `КлиентСервер` | клиент и сервер, `&НаКлиентеНаСервереБезКонтекста` |
| `…Сервер` (явно) | отдельный серверный модуль по смыслу (пример — уточняйте в поставке) |
| `ПовтИсп` | кэш на уровне сеанса, типично для справочных данных; правила актуализации — по **вашему** коду модуля |

**Эвристика:** для кода, общего для формы/клиента/сервера, сначала `*КлиентСервер`; тяжёлую логику — на сервер; `ПовтИсп` — только при явной необходимости кэша.

Пример клиент/сервер: [templates.md](../templates.md#example-client-server)
