# Чеклист: БСП (перед коммитом в зоне библиотеки)

Перед фиксацией проверь **в дополнение** к [coding-standards/checklist-precommit.md](../coding-standards/checklist-precommit.md) (стиль BSL — там).

## Поиск в БСП

- [ ] Перед новой самописной функцией проверена **ваша** выгрузка/ветка: нет ли готового в БСП
- [ ] Просмотрены кандидаты `ОбщегоНазначения` (часто покрывает типовой набор)
- [ ] Для строк — `СтроковыеФункции*`
- [ ] Для кода в обеих средах — `*КлиентСервер` / `ОбщегоНазначенияКлиентСервер` по смыслу

## Вместо дублирования

- [ ] Несколько реквизитов по ссылке — пакетом (`ЗначенияРеквизитовОбъекта` / по соглашениям проекта)
- [ ] Сообщения с привязкой к полю — через `СообщитьПользователю` (где применимо в вашем слое)
- [ ] Подстановки в строки — шаблоны/функции БСП, не «склейка +» на автопилоте
- [ ] Доступ к структуре — `СвойствоСтруктуры` (или иной **ваш** безопасный шаблон)

## Модуль и среда

- [ ] Клиент не тянет лишнего с сервера; серверные модули не вызываются «на клиенте» без смысла
- [ ] `ПовтИсп` — только по назначению, не «на всякий случай»

## Паттерны

- [ ] `ОбработкаПроверкиЗаполнения` согласована с типовыми требованиями
- [ ] Записи в журнал — в рамках соглашений **вашей** поставки (транзакции, уровни, метаданные)
- [ ] Файлы — с учётом `РаботаСФайлами` / политик БСП, не «объект Файл и забыли»
- [ ] Права — `ПравоДоступаОбъекта` или **актуальный** вызов в **вашей** ревизии

## Качество

- [ ] Нет копий функций, которые **уже** есть в БСП с тем же смыслом
- [ ] Самописные функции (если неизбежны) — в **стиле** соседнего кода и БСП (имена, комментарии)
