# Дерево каталогов внешней обработки (EPF)

Ниже — типовая структура **внешней обработки** в XML-представлении. Имена вложенных каталогов форм и макетов соответствуют **именам объектов в метаданных**, а не произвольным меткам.

```
<корень выгрузки внешней обработки>/
├── <ИмяВнешнейОбработки>.xml          # Корневой MetaDataObject → ExternalDataProcessor
├── Ext/
│   ├── ObjectModule.bsl               # Модуль объекта внешней обработки (если есть)
│   ├── Help.xml                       # Метаданные справки (если есть)
│   └── Help/
│       └── <язык>.html               # Текст справки
├── Forms/
│   └── <ИмяФормы>/
│       └── Ext/
│           ├── Form.xml               # Описание управляемой формы (отдельное пространство имён)
│           └── Form/
│               └── Module.bsl         # Модуль управляемой формы
├── Templates/
│   ├── <ИмяМакета>.xml                # Метаданные макета (MetaDataObject)
│   └── <ИмяМакета>/
│       └── Ext/
│           └── Template.<расш>       # Тело: часто Template.xml для табличного документа (MXL)
```

Модуль формы — по навыку [form_patterns](../../form_patterns/SKILL.md). Макеты `Template.xml` — по [mxl_layouts](../../mxl_layouts/SKILL.md).

Пути **`Ext/ObjectModule.bsl`**, **`Forms/...`**, **`Templates/...`** должны быть согласованы с перечислением в **`ChildObjects`** корневого XML (см. [root-metadata-and-forms.md](root-metadata-and-forms.md)).

## Внешний отчёт (ERF): отличия кратко

У **внешнего отчёта** корневой узел метаданных — **`ExternalReport`** вместо **`ExternalDataProcessor`**. Дополнительно в типичном дереве появляются артефакты **системы компоновки данных** и отчётности:

- макеты и описания, связанные со **СКД** (схема компоновки, настройки), часто под **`Templates/`** и/или отдельными XML по соглашению выгрузки;
- **варианты отчёта** — отдельные узлы метаданных отчёта, влияющие на доступные настройки и представление результата.

Точный набор файлов зависит от версии платформы и того, включены ли внешние отчёты с несколькими вариантами и пользовательскими настройками СКД. Логика разделения: **EPF** — обработка без отчётного контура СКД как основного назначения; **ERF** — отчёт с **СКД** и типовым жизненным циклом выполнения отчёта и вариантов.
