# Корневой XML и формы внешнего объекта

## Корневой файл `<Имя>.xml`

Корень документа — **`MetaDataObject`**. Для внешней обработки дочерний элемент задаёт тип **`ExternalDataProcessor`**, для внешнего отчёта — **`ExternalReport`**.

Внутри узла объекта обычно присутствуют:

- **`Properties`** — имя, синоним, комментарий и прочие свойства, включая ссылку на форму по умолчанию;
- **`ChildObjects`** — перечень **реквизитов**, **табличных частей**, **форм**, **макетов** и других дочерних описаний, которые имеют отдельные файлы или внутренние блоки в выгрузке.

### DefaultForm и формы

Свойство вроде **основной (формы по умолчанию)** должно указывать на форму, которая **действительно перечислена** среди форм в **`ChildObjects`** и для которой существует каталог **`Forms/<ИмяФормы>/...`**. Несогласованность имени приводит к ошибке при открытии или загрузке.

### Реквизиты и табличные части в ChildObjects

**Реквизиты** и **табличные части** описываются как дочерние объекты метаданных в **`ChildObjects`** (и при необходимости уточняются вложенными узлами). При добавлении реквизита или ТЧ нужно обновить и **модуль объекта**, и **формы**, которые на них опираются (реквизиты формы, колонки таблиц), иначе получите расхождение между метаданными и UI.

Правила редактирования Form.xml и модуля формы — в навыке **[form_patterns](../../form_patterns/SKILL.md)**; здесь важно помнить, что для внешнего объекта то же разделение «метаданные объекта / форма», но корневой файл — **`ExternalDataProcessor`** или **`ExternalReport`**, а не объект справочника или документа конфигурации.

## Файл формы `Forms/<Имя>/Ext/Form.xml`

Файл **`Form.xml`** описывает **управляемую форму** и использует **пространство имён логической модели формы** (типичный префикс для форм платформы — отличный от корневого **`MetaDataObject`**).

Это принципиально:

- корневой **`…ExternalDataProcessor.xml`** подчинён схеме **классов метаданных** (**MDClasses** и связанные URI платформы);
- **`Form.xml`** подчинён схеме **формы** (часто обозначаемой в материалах платформы как логическая форма / **logform**).

Копировать фрагменты между этими файлами «как есть» нельзя: элементы, атрибуты и namespace различаются. Исправления в **`Form.xml`** выполняйте в терминах **формы**: реквизиты формы, команды, элементы, события — см. **[form_patterns](../../form_patterns/SKILL.md)** и проверки XML по контракту MCP в вашем окружении.

## Модуль объекта

Код общего поведения объекта без привязки к конкретной форме — в **`Ext/ObjectModule.bsl`**. Обработчики команд и серверная логика формы при этом могут находиться в модуле формы; граница та же, что и для объектов конфигурации: не дублировать без нужды и не смешивать контексты клиента и сервера.
