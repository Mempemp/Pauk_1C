# Структура Template.xml (табличный документ)

Индекс навыка — [../SKILL.md](../SKILL.md). Проверка по инструментам среды — [../../../mcp/README.md](../../../mcp/README.md).

## Корень и пространство имён

Корневой элемент документа макета типа **табличный документ** — обычно **`document`** с пространством имён вида **`http://v8.1c.ru/8.2/data/spreadsheet`** и стандартными префиксами для стилей, типов и UI (`style`, `v8`, `v8ui`, `xs`, `xsi`). Конкретный набор деклараций возьмите из **рабочего эталона** вашей платформы или из XSD — не копируйте устаревшие корни между версиями без проверки.

## Порядок узлов под `document`

Логический порядок блоков фиксирован платформой (типичная последовательность):

1. **`languageSettings`** — языки документа.
2. **`columns`** — один или несколько **наборов колонок** (основной без `id`, дополнительные — с **`id`** UUID).
3. **`rowsItem`** — строки и ячейки (повторяющиеся узлы).
4. **`drawing`** — рисунки (опционально, могут повторяться).
5. **`templateMode`**, **`defaultFormatIndex`**, **`height`**, **`vgRows`** — параметры таблицы и режима макета.
6. **`merge`** / **`verticalUnmerge`** — объединения ячеек.
7. **`namedItem`** — именованные области (повторяются).
8. **`line`**, **`font`**, **`format`**, **`picture`** — палитры (повторяются по мере необходимости).

Нарушение порядка или произвольное удаление «пустых» блоков может сделать файл нечитаемым конфигуратором.

## Индексация палитр

- **Шрифты**, **линии**, **`format`** — как правило **плоские списки** в порядке появления; на элементы ссылаются **индексами**.
- Индекс **формата** для ячейки часто **1-based**: `0` означает «формат по умолчанию», ненулевое значение — **N-й** элемент в списке **`format`** (уточняйте по эталону и XSD).

Любое изменение порядка элементов в палитре **сдвигает** все последующие ссылки.

## Колонки: основной и дополнительные наборы

- **Основной набор** `columns` — без атрибута **`id`**: задаёт **`size`**, перечень **`columnsItem`** с **`index`** колонки и ссылкой **`formatIndex`** на ширину/оформление колонки.
- **Дополнительный набор** — тот же тег **`columns`**, но с **`id`** (UUID). Строки макета могут ссылаться на набор через **`columnsID`** в **`row`**, если область использует другую разбивку колонок (шапка и табличная часть с разным числом колонок).

Первый блок **`columns`** без **`id`** — основная сетка; остальные — именованные альтернативные сетки.

## Строки и ячейки

- **`rowsItem`** содержит **`index`** строки (часто **0-based**), опционально **`indexTo`** для диапазона строк с одинаковым содержимым.
- Внутри **`row`**: опционально **`columnsID`** (ссылка на дополнительный набор колонок), **`formatIndex`** высоты строки, **`empty`**, набор **`c`** (группы ячеек).

Каждая группа **`c`** задаёт позицию колонки (**`i`** — индекс, при необходимости) и вложенное содержимое ячейки: **`f`** (индекс формата), **`parameter`**, **`detailParameter`**, блок **`tl`** с локализованным текстом и т.д.

## Именованные области и объединения

- **`namedItem`** связывает **имя области** с диапазоном строк/колонок для вызова **`ТабличныйДокумент.ПолучитьОбласть("Имя")`** в BSL.
- **`merge`** описывает объединённые прямоугольники ячеек; должно быть согласовано с фактической сеткой строк и колонок.

После правок держите в соответствии **имена в `namedItem`**, код заполнения и визуальную структуру ([areas-parameters-and-fill.md](areas-parameters-and-fill.md)).
