# Вложенные транзакции и ТранзакцияАктивна() (E3)

Индекс — в [SKILL.md](../SKILL.md).

## Почему

В платформе вложенный **`НачатьТранзакцию()`** не создаёт «вторую независимую» транзакцию, а увеличивает **счётчик**. **`ОтменитьТранзакцию()`** помечает откат так, что последующий **`ЗафиксироватьТранзакцию()`** на внешнем уровне может привести к ошибке. Внешний код должен **знать**, что внутри был откат.

На практике **предпочтительно** не открывать новую транзакцию в процедуре, которая всегда вызывается уже внутри транзакции вызывающего кода, либо явно документировать контракт (см. ИТС: правила использования транзакций).

### Иллюстрация вложенного вызова (учебный пример)

```bsl
Процедура ЗаписатьДанные(ДанныеДляЗаписи)

    НачатьТранзакцию();
    Попытка
        // ... запись по ДанныеДляЗаписи ...
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации(
            НСтр("ru = 'Запись данных'"),
            УровеньЖурналаРегистрации.Ошибка,,,
            ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
        ВызватьИсключение;
    КонецПопытки;

КонецПроцедуры

Процедура ОбработатьПакетДокументов(МассивДокументов)

    НачатьТранзакцию();
    Попытка
        Для Каждого ДанныеДокумента Из МассивДокументов Цикл
            ЗаписатьДанные(ДанныеДокумента);
        КонецЦикла;
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации(...);
        ВызватьИсключение;
    КонецПопытки;

КонецПроцедуры
```

В реальном коде пакет и «запись одной единицы» часто проектируют так, чтобы **не удваивать** `НачатьТранзакцию` без необходимости.

## ТранзакцияАктивна() — защитно

Имеет смысл, когда нужно **гарантированно** снять активную транзакцию в аварийной ветке (например после вложенной `Попытка` вокруг `Заблокировать`), а не как постоянную «подстраховку» вместо ровной структуры `Попытка` / `Исключение`.

```bsl
Если ТранзакцияАктивна() Тогда
    ОтменитьТранзакцию();
КонецЕсли;
```

## Не маскировать ошибки структуры кода

```bsl
// ПЛОХО: откат только «если активна» вместо явного канона
Попытка
    НачатьТранзакцию();
    ЗафиксироватьТранзакцию();
Исключение
    Если ТранзакцияАктивна() Тогда
        ОтменитьТранзакцию();
    КонецЕсли;
КонецПопытки;

// ПРАВИЛЬНО: канон из transactions-canonical.md
НачатьТранзакцию();
Попытка
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```
