# Исключения, журнал регистрации и пользователь (E1, E7–E9)

Индекс — в [SKILL.md](../SKILL.md).

## E1: Не глотать исключения

Пустой или «молчаливый» `Исключение` оставляет систему в неясном состоянии: данные могли не записаться, в журнале пусто, пользователь не понимает. В обработчике исключения **фиксируйте ошибку** в журнале регистрации (по стандартам ИТС) и дальше либо **пробрасывайте**, либо возвращайте осмысленный результат — по контракту процедуры.

### Канон

```bsl
Попытка
    ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
Исключение
    ИнформацияОбОшибке = ИнформацияОбОшибке();

    ЗаписьЖурналаРегистрации(
        НСтр("ru = 'Проведение документа'"),
        УровеньЖурналаРегистрации.Ошибка,
        ДокументОбъект.Метаданные(),
        ДокументОбъект.Ссылка,
        ПодробноеПредставлениеОшибки(ИнформацияОбОшибке));

    ВызватьИсключение;
КонецПопытки;
```

### Кратко и подробно

| Функция | Куда | Содержимое |
|---------|------|------------|
| `КраткоеПредставлениеОшибки()` | Сообщение пользователю, итог в интерфейсе | Без полного стека |
| `ПодробноеПредставлениеОшибки()` | Журнал регистрации | Стек, вложенные ошибки |

### Два уровня: сервер логирует, клиент показывает

```bsl
Функция ЗаписатьДокумент(ДокументОбъект)
    Попытка
        ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
        Возврат Истина;
    Исключение
        ЗаписьЖурналаРегистрации(
            НСтр("ru = 'Проведение документа'"),
            УровеньЖурналаРегистрации.Ошибка,
            ДокументОбъект.Метаданные(),
            ДокументОбъект.Ссылка,
            ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
        ВызватьИсключение;
    КонецПопытки;
КонецФункции

&НаКлиенте
Процедура ЗаписатьДокумент(Команда)
    Попытка
        ЗаписатьДокументНаСервере();
    Исключение
        ПоказатьПредупреждение(,
            НСтр("ru = 'Не удалось записать документ. Обратитесь к администратору.'"));
    КонецПопытки;
КонецПроцедуры
```

## E7: ЗаписьЖурналаРегистрации

Событие — **строка** (часто иерархия через точку для фильтрации). Уровень — **`УровеньЖурналаРегистрации`**. В комментарий передавайте **контекст** для разбора инцидента (до ограничения платформы на длину строки).

| Уровень | Когда |
|---------|--------|
| `Ошибка` | Операция не выполнена, откат |
| `Предупреждение` | Выполнено с ограничениями |
| `Информация` | Значимые события аудита |
| `Примечание` | Диагностика |

Имя события удобно строить **иерархией** (подсистема.действие.результат) — проще фильтровать в журнале.

```bsl
ИмяСобытия = НСтр("ru = 'ОбменДанными.ОтправкаДанных.Ошибка'");

Комментарий = СтрШаблон(
    НСтр("ru = 'Ошибка при отправке в узел ""%1"". Объектов: %2. Текст: %3'"),
    Строка(УзелОбмена),
    КоличествоОбъектов,
    ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));

ЗаписьЖурналаРегистрации(
    ИмяСобытия,
    УровеньЖурналаРегистрации.Ошибка,
    Метаданные.ПланыОбмена.ОбменСКонтрагентами,
    УзелОбмена,
    Комментарий);
```

Имена метаданных и переменных замените на свои.

## E8: Сообщения пользователю

Нужны: **что случилось простыми словами**, **что сделать**, при необходимости **куда обратиться**. Не выводить сырой стек из общего модуля как единственный текст для пользователя.

Пример: в журнал — подробно, пользователю — кратко и по делу.

```bsl
Попытка
    ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
Исключение
    ЗаписьЖурналаРегистрации(
        НСтр("ru = 'Проведение документа'"),
        УровеньЖурналаРегистрации.Ошибка,
        ДокументОбъект.Метаданные(),
        ДокументОбъект.Ссылка,
        ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));

    ТекстДляПользователя = СтрШаблон(
        НСтр("ru = 'Не удалось провести документ ""%1"". Повторите операцию или обратитесь к администратору. Кратко: %2'"),
        ДокументОбъект,
        КраткоеПредставлениеОшибки(ИнформацияОбОшибке()));

    ВызватьИсключение ТекстДляПользователя;
КонецПопытки;
```

## E9: ВызватьИсключение

| Вызов | Когда |
|-------|--------|
| `ВызватьИсключение;` | Проброс **исходной** ошибки со стеком — типично после ЖР в общем модуле |
| `ВызватьИсключение Текст;` | Новое исключение с понятным текстом — часто на **границе с пользователем**; исходный стек в цепочке не сохраняется как при «голом» пробросе |

```bsl
Процедура ОбработатьДанные(Данные)
    Попытка
        ЗаписатьДанные(Данные);
    Исключение
        ЗаписьЖурналаРегистрации(...);
        ВызватьИсключение;
    КонецПопытки;
КонецПроцедуры

&НаСервере
Процедура ОбработатьНаСервере()
    Попытка
        ОбработатьДанные(ДанныеФормы);
    Исключение
        ВызватьИсключение СтрШаблон(
            НСтр("ru = 'Ошибка обработки данных: %1'"),
            КраткоеПредставлениеОшибки(ИнформацияОбОшибке()));
    КонецПопытки;
КонецПроцедуры
```
