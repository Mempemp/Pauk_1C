# Канонический паттерн транзакции и короткая транзакция (E2, E4)

Индекс — в [SKILL.md](../SKILL.md).

## E2: Почему порядок жёсткий

Транзакция должна быть **атомарной** и **обязательно закрытой** (`ЗафиксироватьТранзакцию` или `ОтменитьТранзакцию`). Долгая открытая транзакция держит блокировки в СУБД: другие сеансы ждут (часто до таймаута), затем получают ошибку ожидания блокировки.

Рекомендации платформы и ИТС: **`НачатьТранзакцию()` сразу перед `Попытка`**, без кода между ними.

### Канонический фрагмент

```bsl
НачатьТранзакцию();
Попытка

    Блокировка = Новый БлокировкаДанных;
    ЭлементБлокировки = Блокировка.Добавить("Документ.РеализацияТоваровУслуг");
    ЭлементБлокировки.УстановитьЗначение("Ссылка", ДокументСсылка);
    Блокировка.Заблокировать();

    ДокументОбъект = ДокументСсылка.ПолучитьОбъект();
    ДокументОбъект.Статус = Перечисления.СтатусыДокументов.Согласован;
    ДокументОбъект.Записать();

    ЗафиксироватьТранзакцию();

Исключение
    ОтменитьТранзакцию();

    ЗаписьЖурналаРегистрации(
        НСтр("ru = 'Согласование документа'"),
        УровеньЖурналаРегистрации.Ошибка,
        Метаданные.Документы.РеализацияТоваровУслуг,
        ДокументСсылка,
        ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));

    ВызватьИсключение;
КонецПопытки;
```

Имена метаданных и перечисления замените на свои.

### Критические требования к порядку

| Требование | Зачем |
|------------|--------|
| `НачатьТранзакцию()` сразу перед `Попытка` | Иначе при ошибке до `Попытка` транзакция может остаться незакрытой |
| `ЗафиксироватьТранзакцию()` — последняя успешная операция в `Попытка` | Код после фиксации при ошибке уже не откатит записанное |
| `ОтменитьТранзакцию()` — **первая** строка в `Исключение` | Сначала откат, потом ЖР и прочее |
| `ЗаписьЖурналаРегистрации` — **после** `ОтменитьТранзакцию()` | Запись в журнал внутри отменённой транзакции может не сохраниться |

### Антипримеры

```bsl
// ПЛОХО: код между НачатьТранзакцию и Попытка
НачатьТранзакцию();
ПодготовитьДанные();
Попытка
    // ...
КонецПопытки;

// ПЛОХО: ЖР до ОтменитьТранзакцию
Исключение
    ЗаписьЖурналаРегистрации(...);
    ОтменитьТранзакцию();
КонецПопытки;

// ПЛОХО: после ЗафиксироватьТранзакцию ещё код в том же Попытка
    ЗафиксироватьТранзакцию();
    ОтправитьОповещение();
Исключение
    ОтменитьТранзакцию();
КонецПопытки;
```

## E4: Минимизировать время транзакции

Пока транзакция открыта, изменённые данные удерживаются в СУБД. **Тяжёлые запросы, циклы обработки, HTTP и ожидание пользователя** — вне транзакции; внутри — только то, что нужно для согласованной записи.

### Правильно

```bsl
МассивДанных = ПодготовитьДанные();
ПроверитьКорректность(МассивДанных);

НачатьТранзакцию();
Попытка
    Для Каждого ДанныеСтроки Из МассивДанных Цикл
        ЗаписатьСтроку(ДанныеСтроки);
    КонецЦикла;
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ЗаписьЖурналаРегистрации(...);
    ВызватьИсключение;
КонецПопытки;
```

### Неправильно

```bsl
// ПЛОХО: долгий запрос, обработка и HTTP внутри одной транзакции
НачатьТранзакцию();
Попытка
    Результат = ВыполнитьСложныйЗапрос();
    Для Каждого Строка Из Результат Цикл
        ОбработатьСтроку(Строка);
    КонецЦикла;
    ОтправитьHTTPЗапрос(Данные);
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
КонецПопытки;
```
