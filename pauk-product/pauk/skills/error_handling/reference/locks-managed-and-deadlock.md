# Управляемые блокировки, объект и порядок (E5, E6, E11)

Индекс — в [SKILL.md](../SKILL.md).

## E5: БлокировкаДанных перед чтением

### Почему

Нужно исключить ситуацию, когда два сеанса **читают одни и те же остатки**, оба считают, что списание возможно, и оба записывают — итог неверный. Схема: **сначала заблокировать нужные измерения**, затем читать и менять.

Порядок «прочитал без блокировки — потом записал» даёт **гонку** между чтением и записью.

```
Без блокировки:  сеанс A читает остаток 10, сеанс B читает 10;
                 оба решают списать; суммарно списали больше, чем было.

С блокировкой:  один сеанс держит блокировку, читает, меняет, фиксирует;
                 второй ждёт и видит уже обновлённые данные.
```

### Паттерн: заблокировать — прочитать — изменить — записать

Имена регистра и параметров — из примера; в конфигурации подставьте свои.

```bsl
НачатьТранзакцию();
Попытка

    Блокировка = Новый БлокировкаДанных;
    ЭлементБлокировки = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
    ЭлементБлокировки.УстановитьЗначение("Номенклатура", НоменклатураСсылка);
    ЭлементБлокировки.УстановитьЗначение("Склад", СкладСсылка);
    ЭлементБлокировки.Режим = РежимБлокировкиДанных.Исключительный;
    Блокировка.Заблокировать();

    Запрос = Новый Запрос;
    Запрос.Текст =
    "ВЫБРАТЬ
    |   Остатки.КоличествоОстаток КАК Остаток
    |ИЗ
    |   РегистрНакопления.ТоварыНаСкладах.Остатки(,
    |       Номенклатура = &Номенклатура И Склад = &Склад) КАК Остатки";
    Запрос.УстановитьПараметр("Номенклатура", НоменклатураСсылка);
    Запрос.УстановитьПараметр("Склад", СкладСсылка);

    Результат = Запрос.Выполнить();
    Если Результат.Пустой() Тогда
        ВызватьИсключение НСтр("ru = 'Нет остатков на складе.'");
    КонецЕсли;

    Выборка = Результат.Выбрать();
    Выборка.Следующий();

    Если Выборка.Остаток < ТребуемоеКоличество Тогда
        ВызватьИсключение СтрШаблон(
            НСтр("ru = 'Недостаточно остатков. На складе: %1, требуется: %2.'"),
            Выборка.Остаток, ТребуемоеКоличество);
    КонецЕсли;

    // ... запись движений по результату проверки ...

    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ЗаписьЖурналаРегистрации(...);
    ВызватьИсключение;
КонецПопытки;
```

Тексты сложных запросов, соединения и функции вроде `ЕСТЬNULL` — в отдельном навыке по **запросам**.

## E6: ЗаблокироватьДанныеДляРедактирования

**Пессимистичная** блокировка объекта: второй пользователь при попытке изменить тот же объект получит сообщение о занятости. Удобно против **потери последней записи** (lost update) при параллельном редактировании.

Для формы объекта платформа часто уже управляет блокировкой; ручной вызов — при **программном** изменении ссылки на объект вне стандартной формы.

```bsl
Процедура ИзменитьСтатусДокумента(ДокументСсылка, НовыйСтатус)

    НачатьТранзакцию();
    Попытка
        ЗаблокироватьДанныеДляРедактирования(ДокументСсылка);
        ДокументОбъект = ДокументСсылка.ПолучитьОбъект();
        ДокументОбъект.Статус = НовыйСтатус;
        ДокументОбъект.Записать();
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации(
            НСтр("ru = 'Изменение статуса документа'"),
            УровеньЖурналаРегистрации.Ошибка,,,
            ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
        ВызватьИсключение;
    КонецПопытки;

КонецПроцедуры
```

### Сравнение типов

| Механизм | Уровень | Когда |
|----------|---------|--------|
| `БлокировкаДанных` | Управляемая, по наборам записей | Остатки, атомарные операции над измерениями |
| `ЗаблокироватьДанныеДляРедактирования` | Объект целиком | Параллельное редактирование одной ссылки |
| Автоматические блокировки при записи | Зависят от режима блокировок | Всегда в игре; настройка конфигурации |

## E11: Единый порядок и взаимная блокировка

Если сеанс A держит ресурс 1 и ждёт 2, а сеанс B держит 2 и ждёт 1, СУБД может выбрать жертву и откатить одну транзакцию — это **взаимная блокировка** (в англоязычной литературе deadlock).

**Практика:** блокировать в **одинаковом порядке** (например отсортировать ссылки по возрастанию и блокировать по очереди).

```bsl
МассивСсылок = ОбщегоНазначенияКлиентСервер.СвернутьМассив(МассивДокументов);
МассивСсылок.СортироватьПоЗначению();

Для Каждого Ссылка Из МассивСсылок Цикл
    ЗаблокироватьДанныеДляРедактирования(Ссылка);
КонецЦикла;
```

Имя `ОбщегоНазначенияКлиентСервер` — из типовых конфигураций; замените на проектный API.

### Ошибка при Заблокировать()

Внутренняя `Попытка` вокруг `Блокировка.Заблокировать()` позволяет перевести таймаут в понятное сообщение пользователю. Во внешнем `Исключение` после внутреннего отката транзакция может уже быть снята — отсюда иногда проверка **`ТранзакцияАктивна()`** перед `ОтменитьТранзакцию()` (см. [transactions-nested.md](transactions-nested.md)); согласуйте с каноном вашего проекта.

```bsl
НачатьТранзакцию();
Попытка
    Блокировка = Новый БлокировкаДанных;
    ЭлементБлокировки = Блокировка.Добавить("Справочник.Номенклатура");
    ЭлементБлокировки.УстановитьЗначение("Ссылка", НоменклатураСсылка);

    Попытка
        Блокировка.Заблокировать();
    Исключение
        ОтменитьТранзакцию();
        ВызватьИсключение СтрШаблон(
            НСтр("ru = 'Не удалось заблокировать ""%1"". Данные заняты другим пользователем. Повторите попытку позже.'"),
            НоменклатураСсылка);
    КонецПопытки;

    НоменклатураОбъект = НоменклатураСсылка.ПолучитьОбъект();
    НоменклатураОбъект.Записать();

    ЗафиксироватьТранзакцию();
Исключение
    Если ТранзакцияАктивна() Тогда
        ОтменитьТранзакцию();
    КонецЕсли;
    ЗаписьЖурналаРегистрации(...);
    ВызватьИсключение;
КонецПопытки;
```
