# Вызовы с формы на сервер: директивы и «всё за один раз» (F1–F2)

Индекс и приоритеты — в [SKILL.md](../SKILL.md).

## Простыми словами

- **Клиент и сервер** — интерфейс (форма) выполняется у пользователя на **клиенте**, а чтение и запись в базу — на **сервере приложений**. Когда из модуля формы вызывается процедура с директивой «на сервер», платформа пересылает на сервер параметры и при необходимости **всю форму целиком** — это занимает время и сеть. Имеет смысл не злоупотреблять лишними такими «поездками».
- **Один вызов вместо нескольких** — если при смене контрагента нужно подставить договор, валюту и курс, не делайте три отдельные серверные функции подряд; напишите **одну**, которая по переданным ссылке и дате вернёт **структуру со всеми полями**. Пользователь меньше ждёт, сервер реже дергается.

(В чужой документации это иногда называют «батчингом» или «агрегацией вызовов» — по смыслу это одно и то же: **собрать данные за одно обращение**.)

## Напоминание

Каждый **`&НаСервере`** — платформа **пересылает на сервер и обратно весь контекст формы** (реквизиты, табличные части на форме и т.д.). Для больших ТЧ это главный источник объёма и задержки.

**Практика:** по возможности выносить чтение из БД в **`&НаСервереБезКонтекста`**, передавая **ссылки и примитивы**; объединять несколько логических запросов в **один** серверный метод, возвращающий структуру или набор полей.

## Порядок предпочтения директив (от лучшего к худшему для нагрузки)

1. `&НаКлиентеНаСервереБезКонтекста` — чистые вычисления, без БД и без данных формы на сервере в смысле контекста.
2. `&НаСервереБезКонтекста` — обращение к БД **без** контекста формы.
3. `&НаСервере` — нужен доступ к реквизитам формы на сервере как к контексту.
4. `&НаКлиенте` — интерактив, вызов серверных процедур из формы.

## Правило принятия решения

```
Нужно ли обращаться к базе данных?
├── Нет → &НаКлиенте или &НаКлиентеНаСервереБезКонтекста
└── Да →
    Нужен ли доступ к реквизитам формы на сервере (чтение/изменение контекста формы)?
    ├── Нет → &НаСервереБезКонтекста (передаём только параметры)
    └── Да → &НаСервере (передаётся контекст формы)
```
