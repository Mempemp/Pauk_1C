# Привязка обработчиков в `Form.xml` и имена процедур

Индекс навыка — в [../SKILL.md](../SKILL.md). Порядок вызова обработчиков на стороне модуля — в [form-events-lifecycle.md](form-events-lifecycle.md).

---

## Зачем это нужно

Только процедура в `Form.Module.bsl` **не подключится** к событию формы, если в **`Form.xml`** (обычно в каталоге выше файла модуля) в узле **`События`** / **`Events`** не указано имя метода. После добавления обработчика в модуль **проверьте XML** формы.

---

## Фрагмент `Form.xml`

```xml
<Events>
	<Event name="OnOpen">ПриОткрытии</Event>
	<Event name="BeforeWrite">ПередЗаписью</Event>
	<Event name="OnCreateAtServer">ПриСозданииНаСервере</Event>
</Events>
```

Значение внутри тега `Event` — это **имя процедуры** в модуле формы, объявленной с нужными директивами компиляции.

Имена атрибутов (`name`) могут в выгрузке быть на английском; в EDT встречаются русские обозначения — ориентируйтесь на **схему** вашей выгрузки и на синтакс-помощник платформы.

---

## Соответствие: имя события в XML → типичное имя обработчика (рус.)

| Имя в XML (пример) | Типичное имя процедуры | Где выполняется |
|--------------------|------------------------|-----------------|
| `OnOpen` | `ПриОткрытии` | клиент |
| `OnClose` | `ПриЗакрытии` | клиент |
| `BeforeWrite` | `ПередЗаписью` | клиент (отмена записи через `Отказ`) |
| `AfterWrite` | `ПослеЗаписи` | клиент |
| `OnCreateAtServer` | `ПриСозданииНаСервере` | сервер |
| `BeforeWriteAtServer` | `ПередЗаписьюНаСервере` | сервер |
| `AfterWriteAtServer` | `ПослеЗаписиНаСервере` | сервер |
| `OnReadAtServer` | `ПриЧтенииНаСервере` | сервер |

Точные имена событий и сигнатуры — в документации платформы для вашей версии; для порядка вызовов см. [form-events-lifecycle.md](form-events-lifecycle.md).
