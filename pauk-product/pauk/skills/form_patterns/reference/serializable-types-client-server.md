# Типы данных между клиентом и сервером (F12)

Индекс — в [SKILL.md](../SKILL.md).

## Почему

Не все значения можно передать между клиентом и сервером. Ошибки вида «значение не может быть сериализовано» устраняются выбором переносимых типов.

## Обычно передаются без проблем

| Тип | Пример |
|-----|--------|
| Примитивы | Строка, Число, Дата, Булево, Неопределено, NULL |
| Ссылки | `СправочникСсылка.*`, `ДокументСсылка.*` |
| Перечисления | `ПеречислениеСсылка.*` |
| Коллекции | `Структура`, `Соответствие`, `Массив` |
| Фиксированные коллекции | `ФиксированнаяСтруктура`, `ФиксированныйМассив` |
| Хранилище значения | `ХранилищеЗначения` |

## Не передавать как параметры/результат границы клиент–сервер

| Тип | Замечание | Альтернатива |
|-----|-----------|--------------|
| `ТаблицаЗначений` | Серверный контейнер | Массив структур, ДанныеФормы*, выборка в переносимые структуры |
| `ДеревоЗначений` | Аналогично | Данные формы, своя структура |
| `ОбъектМетаданных` | Не сериализуется | Строка с именем, идентификатор |
| `COMОбъект` | Не передаётся между клиентом и сервером | — |
| `Запрос`, `РезультатЗапроса` | Контекст выполнения | Передавать уже материализованные данные |

Уточняйте полный список для вашей версии платформы в синтакс-помощнике и документации ИТС.

## Пример

```bsl
&НаСервереБезКонтекста
Функция ПолучитьДанныеНаСервере(СсылкаНаОбъект)
    Возврат Новый Структура("Наименование, Код",
        СсылкаНаОбъект.Наименование, СсылкаНаОбъект.Код);
КонецФункции
```

```bsl
// ПЛОХО: ТаблицаЗначений не уйдёт на клиент как результат границы вызова
&НаСервереБезКонтекста
Функция ПолучитьТаблицуНаСервере()
    Возврат Новый ТаблицаЗначений;
КонецФункции
```
