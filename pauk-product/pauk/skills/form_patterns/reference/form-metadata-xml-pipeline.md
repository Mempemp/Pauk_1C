# Генерация и правка формы на уровне метаданных и XML

Индекс навыка — в [../SKILL.md](../SKILL.md). Контракт **имён инструментов** MCP (поиск форм, разбор макета, XSD, проверка XML, компиляция) — в [README MCP](../../../mcp/README.md): не подставляйте имена «из памяти», используйте только подключённые серверы.

---

## Общий подход

1. Найти в конфигурации **близкий по смыслу** пример формы (поиск по метаданным / коду — через доступные инструменты или точечное чтение выгрузки).
2. Изучить **иерархию элементов**, привязки к данным, реквизиты, команды и заявленные события (структура `Form.xml` + модуль).
3. Получить **схему** допустимой структуры формы для правок (XSD или эквивалент в вашей цепочке) и не выходить за её рамки.
4. Сформировать или изменить **`Form.xml`**, затем **проверить** против схемы, исправить ошибки.
5. Выполнить **сборку/загрузку** формы в согласованной с проектом среде (скрипты, CI, MCP — что реально подключено).

Тяжёлые пакетные операции с метаданными и внешними утилитами выносите в **отдельный контролируемый шаг** (не смешивайте с правкой только модуля формы, если задача этого не требует).

---

## Новая форма (черновик этапов)

- Поиск аналога → разбор структуры аналога → валидация целевой структуры XML → правка/генерация → проверка → загрузка в ИБ или выгрузку по процессу проекта.

## Существующая форма

- Снять текущую структуру (элементы, привязки, команды, события) → свериться со схемой → внести изменения в XML → проверка → применить через принятый в проекте инструмент.

После изменения XML не забудьте **согласовать** обработчики в [form-xml-events.md](form-xml-events.md).
