# Видимость, табличные части на форме, оповещения (F9–F11)

Индекс — в [SKILL.md](../SKILL.md).

## F9: управление видимостью и доступностью

### Почему

Много отдельных серверных вызовов «по одному элементу» дают задержки и дёрганье UI. Лучше **одна** процедура, которая выставляет все флаги видимости и доступности.

### Правильно

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    УправлениеВидимостью();
КонецПроцедуры

&НаСервере
Процедура УправлениеВидимостью()

    ЭтоУслуга = Объект.ВидОперации = Перечисления.ВидыОпераций.Услуга;

    Элементы.Склад.Видимость = НЕ ЭтоУслуга;
    Элементы.ТоварыКоличество.Видимость = НЕ ЭтоУслуга;
    Элементы.ТоварыЕдиницаИзмерения.Видимость = НЕ ЭтоУслуга;
    Элементы.ГруппаДоставка.Видимость = НЕ ЭтоУслуга;

    Элементы.Контрагент.ТолькоПросмотр = Объект.Проведен;

КонецПроцедуры

&НаКлиенте
Процедура ВидОперацииПриИзменении(Элемент)
    УправлениеВидимостью();
КонецПроцедуры
```

### Неправильно

```bsl
// ПЛОХО: несколько последовательных вызовов на сервер ради видимости
&НаКлиенте
Процедура ВидОперацииПриИзменении(Элемент)
    УстановитьВидимостьСклада();
    УстановитьВидимостьКоличества();
    ...
КонецПроцедуры
```

## F10: табличная часть на форме

### Почему

Типичная ошибка — **серверный вызов в цикле** по строкам или лишние round-trip на каждое поле. Пересчёт сумм и агрегатов по строке делайте **на клиенте**, если достаточно уже загруженных данных; к серверу — **один** вызов за пакет данных по номенклатуре и т.п.

### Пересчёт строки на клиенте

```bsl
&НаКлиенте
Процедура ТоварыКоличествоПриИзменении(Элемент)
    ТекущаяСтрока = Элементы.Товары.ТекущиеДанные;
    ТекущаяСтрока.Сумма = РассчитатьСумму(
        ТекущаяСтрока.Количество, ТекущаяСтрока.Цена, ТекущаяСтрока.СтавкаНДС);
КонецПроцедуры

&НаКлиенте
Процедура ТоварыНоменклатураПриИзменении(Элемент)
    ТекущаяСтрока = Элементы.Товары.ТекущиеДанные;
    ДанныеНоменклатуры = ПолучитьДанныеНоменклатуры(ТекущаяСтрока.Номенклатура);

    ТекущаяСтрока.ЕдиницаИзмерения = ДанныеНоменклатуры.ЕдиницаИзмерения;
    ТекущаяСтрока.Цена = ДанныеНоменклатуры.Цена;
    ТекущаяСтрока.СтавкаНДС = ДанныеНоменклатуры.СтавкаНДС;
    ТекущаяСтрока.Сумма = РассчитатьСумму(
        ТекущаяСтрока.Количество, ТекущаяСтрока.Цена, ТекущаяСтрока.СтавкаНДС);
КонецПроцедуры
```

`РассчитатьСумму` / `ПолучитьДанныеНоменклатуры` — ваши процедуры (`&НаКлиентеНаСервереБезКонтекста` и `&НаСервереБезКонтекста` по смыслу).

### Обход ТЧ на клиенте

```bsl
&НаКлиенте
Процедура ПересчитатьИтого()
    ИтогоСумма = 0;
    Для Каждого СтрокаТовара Из Объект.Товары Цикл
        ИтогоСумма = ИтогоСумма + СтрокаТовара.Сумма;
    КонецЦикла;
    ИтогоПоДокументу = ИтогоСумма;
КонецПроцедуры
```

`ИтогоПоДокументу` — пример реквизита формы для отображения; имя и привязка к элементам — по вашей форме.

## F11: оповещения между формами

### Почему

После записи или другого значимого действия открытые формы часто должны обновить списки или поля. Используйте **`Оповестить`** и **`ОбработкаОповещения`**.

### Паттерн

```bsl
&НаКлиенте
Процедура ПослеЗаписи(ПараметрыЗаписи)
    Оповестить("Запись_РеализацияТоваровУслуг",
        Новый Структура("Ссылка, Проведен", Объект.Ссылка, Объект.Проведен),
        ЭтотОбъект);
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)

    Если ИмяСобытия = "Запись_РеализацияТоваровУслуг" Тогда
        Элементы.Список.Обновить();
    КонецЕсли;

КонецПроцедуры
```

Строковые имена событий согласуйте в команде (единый префикс или регистр имён).
