# Чеклист: модуль управляемой формы

Используйте вместе с общим [checklist-precommit.md](../coding-standards/checklist-precommit.md) из **coding-standards**.

## Серверные вызовы

- [ ] Где возможно, `&НаСервереБезКонтекста` вместо `&НаСервере` без необходимости полного контекста формы
- [ ] Нет лишней цепочки отдельных серверных вызовов при одном действии пользователя — по возможности один вызов, который возвращает все нужные поля сразу
- [ ] Серверная функция возвращает все нужные поля за один раз
- [ ] Нет серверных вызовов внутри цикла по строкам ТЧ

## Преобразование данных формы

- [ ] После `РеквизитФормыВЗначение` для изменяемого объекта есть `ЗначениеВРеквизитФормы`
- [ ] Преобразование применяется там, где действительно нужны методы объекта или контракт общего модуля
- [ ] Значения, которые передаются между клиентом и сервером, — из допустимых для такой передачи типов (см. навык, раздел про типы)

## Диалоги и навигация

- [ ] Вопросы и предупреждения — через асинхронные методы с `ОписаниеОповещения`
- [ ] Нет запрещённых в веб-клиенте модальных вызовов (`Вопрос`, `Предупреждение`, модальное открытие формы)
- [ ] Открытие форм — `ОткрытьФорму` с оповещением при необходимости результата
- [ ] Обработчики оповещений с суффиксом `Экспорт` там, где требует платформа

## Обработчики событий

- [ ] Инициализация формы и тяжёлые настройки списков — в `ПриСозданииНаСервере` по смыслу задачи
- [ ] Проверки с диалогом до записи — корректный клиентский обработчик (например `ПередЗаписью` / `ПередЗаписьюНаКлиенте` в зависимости от версии и шаблона формы)
- [ ] Логика после записи объекта — в серверных обработчиках записи по правилам платформы
- [ ] Оповещение других форм — в подходящем месте (часто `ПослеЗаписи` на клиенте)

## Производительность и списки

- [ ] Изменения видимости и доступности сгруппированы в одну серверную процедуру, где это уместно
- [ ] Пересчёты по строке ТЧ — на клиенте, если данных на клиенте достаточно
- [ ] Динамические списки: отборы через параметры и компоновку, а не только жёсткий текст запроса без учёта настроек
- [ ] Условное оформление для динамических условий настроено программно там, где нужно
