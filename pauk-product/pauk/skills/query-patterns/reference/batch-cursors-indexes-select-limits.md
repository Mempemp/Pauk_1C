# Пакет через таблицу значений, выборка и выгрузка, индексы, поля результата и лимиты

## Правило 6: пакет через ТаблицаЗначений вместо N запросов

**Почему:** массив строк из кода можно передать параметром в запрос и соединить с метаданными одним выполнением.

### Правильно

```bsl
ТаблицаДанных = Новый ТаблицаЗначений;
ТаблицаДанных.Колонки.Добавить("Штрихкод", Новый ОписаниеТипов("Строка", , Новый КвалификаторыСтроки(13)));
ТаблицаДанных.Колонки.Добавить("Количество", Новый ОписаниеТипов("Число", Новый КвалификаторыЧисла(15, 3)));

Для Каждого СтрокаИмпорта Из ДанныеИмпорта Цикл
    НоваяСтрока = ТаблицаДанных.Добавить();
    НоваяСтрока.Штрихкод = СтрокаИмпорта.Штрихкод;
    НоваяСтрока.Количество = СтрокаИмпорта.Количество;
КонецЦикла;

Запрос = Новый Запрос;
Запрос.УстановитьПараметр("ДанныеИмпорта", ТаблицаДанных);

Запрос.Текст =
"ВЫБРАТЬ
|   Данные.Штрихкод КАК Штрихкод,
|   Данные.Количество КАК Количество
|ПОМЕСТИТЬ втДанныеИмпорта
|ИЗ
|   &ДанныеИмпорта КАК Данные
|;
|
|////////////////////////////////////////////////////////////////////////////////
|ВЫБРАТЬ
|   Штрихкоды.Номенклатура КАК Номенклатура,
|   втДанные.Количество КАК Количество
|ИЗ
|   втДанныеИмпорта КАК втДанные
|       ВНУТРЕННЕЕ СОЕДИНЕНИЕ РегистрСведений.ШтрихкодыНоменклатуры КАК Штрихкоды
|       ПО втДанные.Штрихкод = Штрихкоды.Штрихкод";

Результат = Запрос.Выполнить();
```

### Неправильно

```bsl
// ПЛОХО: поиск по каждой строке отдельным запросом
Для Каждого СтрокаИмпорта Из ДанныеИмпорта Цикл

    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Штрихкоды.Номенклатура ИЗ РегистрСведений.ШтрихкодыНоменклатуры КАК Штрихкоды ГДЕ Штрихкоды.Штрихкод = &Штрихкод";
    Запрос.УстановитьПараметр("Штрихкод", СтрокаИмпорта.Штрихкод);

КонецЦикла;
```

## Правило 7: выборка vs выгрузка

**Почему:** курсор (`Выбрать` / `Следующий`) экономит память при последовательном обходе; `Выгрузить()` загружает весь результат — удобно при небольшом объёме и произвольном доступе к строкам.

### Выборка

```bsl
Результат = Запрос.Выполнить();
Выборка = Результат.Выбрать();

Пока Выборка.Следующий() Цикл
    Сообщение = СтрШаблон("Товар: %1, Остаток: %2",
        Выборка.Наименование, Выборка.Остаток);
КонецЦикла;
```

### Двухуровневая выборка (группировки)

```bsl
Результат = Запрос.Выполнить();
ВыборкаКонтрагенты = Результат.Выбрать(ОбходРезультатаЗапроса.ПоГруппировкам);

Пока ВыборкаКонтрагенты.Следующий() Цикл

    ИтогоПоКонтрагенту = ВыборкаКонтрагенты.Сумма;

    ВыборкаДокументы = ВыборкаКонтрагенты.Выбрать();
    Пока ВыборкаДокументы.Следующий() Цикл
        ОбработатьДокумент(ВыборкаДокументы.ДокументСсылка, ВыборкаДокументы.Сумма);
    КонецЦикла;

КонецЦикла;
```

### Выгрузка

```bsl
// Вся таблица в памяти: ок для ограниченного объёма, поиска, передачи в другую процедуру
ТаблицаРезультат = Запрос.Выполнить().Выгрузить();
НайденнаяСтрока = ТаблицаРезультат.Найти(ИскомаяНоменклатура, "Номенклатура");
```

## Правило 8: индексация и помощь оптимизатору

**Почему:** для соединений и частых фильтров нужны индексируемые пути; для временных таблиц — `ИНДЕКСИРОВАТЬ ПО` по полям JOIN.

### Временная таблица и соединение с остатками

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   ДанныеЗаказов.Номенклатура КАК Номенклатура,
|   ДанныеЗаказов.Количество КАК Количество
|ПОМЕСТИТЬ втЗаказы
|ИЗ
|   &ТаблицаЗаказов КАК ДанныеЗаказов
|
|ИНДЕКСИРОВАТЬ ПО
|   Номенклатура
|;
|
|////////////////////////////////////////////////////////////////////////////////
|ВЫБРАТЬ
|   втЗаказы.Номенклатура,
|   втЗаказы.Количество КАК Заказано,
|   ЕСТЬNULL(Остатки.КоличествоОстаток, 0) КАК НаСкладе
|ИЗ
|   втЗаказы
|       ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Остатки(,
|           Номенклатура В (ВЫБРАТЬ втЗаказы.Номенклатура ИЗ втЗаказы)) КАК Остатки
|       ПО втЗаказы.Номенклатура = Остатки.Номенклатура";
```

### Конфигурация

Для постоянных полей: в конфигураторе/EDT свойство реквизита «Индексировать», если по полю часто фильтруют и это оправдано моделью данных.

## Правило 9: только нужные поля

**Почему:** лишние поля увеличивают трафик и нагрузку; `Таблица.*` тянет тяжёлые типы без необходимости и мешает покрывающим стратегиям доступа.

### Правильно

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   Контрагенты.Ссылка,
|   Контрагенты.Наименование,
|   Контрагенты.ИНН
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты
|ГДЕ
|   Контрагенты.ЭтоГруппа = ЛОЖЬ";
```

### Неправильно

```bsl
// ПЛОХО: все реквизиты, в т.ч. тяжёлые
Запрос.Текст =
"ВЫБРАТЬ
|   Контрагенты.*
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты";
```

## Правило 10: ограничение результата

**Почему:** неограниченная выборка может исчерпать память и время; для UI часто достаточно первых N строк при явной сортировке.

### ПЕРВЫЕ N

```bsl
Запрос.Текст =
"ВЫБРАТЬ ПЕРВЫЕ 100
|   Товары.Наименование,
|   Товары.Код
|ИЗ
|   Справочник.Номенклатура КАК Товары
|ГДЕ
|   Товары.Наименование ПОДОБНО &СтрокаПоиска
|УПОРЯДОЧИТЬ ПО
|   Товары.Наименование";

Запрос.УстановитьПараметр("СтрокаПоиска", СтрокаПоиска + "%");
```

### Динамические списки

Для данных в динамическом списке пагинацию и дозагрузку обеспечивает платформа; не дублируйте свою пагинацию без необходимости.
