# РАЗЛИЧНЫЕ, ВЫРАЗИТЬ по составным типам и фильтры при ЛЕВОМ СОЕДИНЕНИИ

## Правило 11: не маскируйте дубликаты через РАЗЛИЧНЫЕ

**Почему:** `РАЗЛИЧНЫЕ` вынуждает устранять дубликаты на всём наборе; если дубликаты из лишнего JOIN — исправляйте соединение и гранулярность строк.

### Правильно — устранить причину

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   Контрагенты.Ссылка,
|   Контрагенты.Наименование
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты
|ГДЕ
|   Контрагенты.Ссылка В
|       (ВЫБРАТЬ РАЗЛИЧНЫЕ Реализация.Контрагент
|        ИЗ Документ.РеализацияТоваровУслуг КАК Реализация
|        ГДЕ Реализация.Дата >= &ДатаНачала)";
```

### Неправильно — РАЗЛИЧНЫЕ как латка после лишнего JOIN

```bsl
// ПЛОХО: раздувание строк и последующая дедупликация
Запрос.Текст =
"ВЫБРАТЬ РАЗЛИЧНЫЕ
|   Контрагенты.Наименование
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты
|       СОЕДИНЕНИЕ Документ.РеализацияТоваровУслуг КАК Реализация
|       ПО Контрагенты.Ссылка = Реализация.Контрагент";
```

## Правило 12: ВЫРАЗИТЬ для составных типов

**Почему:** обращение к реквизиту через поле составного типа без сужения типа заставляет платформу учитывать все варианты типа. `ВЫРАЗИТЬ(... КАК Документ.Имя)` и отбор `ССЫЛКА` сужают набор соединений.

**Стандарт ИТС:** использование `ВЫРАЗИТЬ` в запросах.

### Правильно

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   Движения.Период,
|   ВЫРАЗИТЬ(Движения.Регистратор КАК Документ.РеализацияТоваровУслуг).Контрагент КАК Контрагент,
|   Движения.Количество
|ИЗ
|   РегистрНакопления.ТоварыНаСкладах КАК Движения
|ГДЕ
|   Движения.Регистратор ССЫЛКА Документ.РеализацияТоваровУслуг";
```

### Неправильно

```bsl
// ПЛОХО: Регистратор.Реквизит по всему составному типу
Запрос.Текст =
"ВЫБРАТЬ
|   Движения.Период,
|   Движения.Регистратор.Контрагент КАК Контрагент,
|   Движения.Количество
|ИЗ
|   РегистрНакопления.ТоварыНаСкладах КАК Движения";
```

## Правило 13: условие соединения vs ГДЕ в LEFT JOIN

**Почему:** условие в `ПО` ограничивает правую сторону до соединения; перенос фильтра по правой таблице в `ГДЕ` отбрасывает строки с NULL и превращает LEFT в фактический INNER.

### Правильно — фильтр в параметрах ВТ / в ПО

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   Номенклатура.Наименование,
|   ЕСТЬNULL(Цены.Цена, 0) КАК Цена
|ИЗ
|   Справочник.Номенклатура КАК Номенклатура
|       ЛЕВОЕ СОЕДИНЕНИЕ РегистрСведений.ЦеныНоменклатуры.СрезПоследних(&ДатаЦен,
|           ВидЦены = &ВидЦены) КАК Цены
|       ПО Номенклатура.Ссылка = Цены.Номенклатура";
```

### Неправильно

```bsl
// ПЛОХО: фильтр по правой части в ГДЕ — теряются товары без цены
Запрос.Текст =
"ВЫБРАТЬ
|   Номенклатура.Наименование,
|   Цены.Цена
|ИЗ
|   Справочник.Номенклатура КАК Номенклатура
|       ЛЕВОЕ СОЕДИНЕНИЕ РегистрСведений.ЦеныНоменклатуры.СрезПоследних КАК Цены
|       ПО Номенклатура.Ссылка = Цены.Номенклатура
|ГДЕ
|   Цены.ВидЦены = &ВидЦены";
```
