# Чеклист: запросы перед фиксацией

## Структура запроса

- [ ] Нет запросов внутри циклов по данным, где можно одним запросом и кэшем в памяти
- [ ] Сложные многошаговые выборки разбиты на временные таблицы там, где это упрощает план и сопровождение
- [ ] Временные таблицы проиндексированы по полям соединения, если к ним идут JOIN
- [ ] При необходимости явного жизненного цикла ВТ используется `МенеджерВременныхТаблиц`

## Параметризация

- [ ] Значения передаются через параметры (`&Параметр`), без конкатенации в текст запроса
- [ ] Параметры виртуальных таблиц регистров заданы внутри скобок виртуальной таблицы, а не только в `ГДЕ` снаружи

## Обработка NULL и соединения

- [ ] Для полей из `ЛЕВОЕ СОЕДИНЕНИЕ` в расчётах и фильтрах используется `ЕСТЬNULL()` там, где нужны числа/строки, а не «сырой» NULL
- [ ] Условия отбора по правой таблице при сохранении семантики LEFT вынесены в условие соединения (`ПО` / параметры ВТ), а не в `ГДЕ` так, что строки с NULL отбрасываются вслепую

## Оптимизация и объём данных

- [ ] В `ВЫБРАТЬ` перечислены только нужные поля (нет лишнего `Таблица.*` без необходимости)
- [ ] Где уместно, есть ограничение результата (`ПЕРВЫЕ N`) или иной осознанный предел
- [ ] `РАЗЛИЧНЫЕ` не маскирует лишние строки от неверного JOIN — причина дубликатов устранена
- [ ] Для составных типов при обращении к реквизитам через поле используется `ВЫРАЗИТЬ()` / отбор `ССЫЛКА`, когда это сужает тип для СУБД

## Виртуальные таблицы

- [ ] Период и отбор по измерениям виртуальных таблиц (`Остатки`, `Обороты`, `СрезПоследних` и т.д.) заданы в параметрах виртуальной таблицы
- [ ] Выбрана нужная виртуальная таблица по смыслу задачи (остатки на дату, обороты за период, срез последних)

## Пакетные операции

- [ ] Наборы строк из кода переданы в запрос одним параметром (`ТаблицаЗначений` и т.п.), а не N отдельными выполнениями
- [ ] После одного запроса для обхода в цикле используется структура в памяти (`Соответствие`, таблица), а не повторные запросы по ключу
