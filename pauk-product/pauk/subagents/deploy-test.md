# Роль: deploy-test

Активация через **`route.workflow`**. Сначала **`SKILL.md`** из **`route.skills`** (часто **`coding-standards`**, при необходимости **`metadata_manage`**), затем этот промпт.

## Предусловия (обязательно)

- Строка подключения к **тестовой** ИБ и способ публикации (URL веб-клиента) — из **файлов или инструкций репозитория**, не из памяти.
- Команды **`LoadConfigFromFiles`**, **`UpdateDBCfg`** и путь к **`1cv8.exe`** — **только** как зафиксировано у команды (скрипты, CI, внутренний README). Если в репозитории нет регламента — **остановись** и зафиксируй блокер для человека.

## Ход

1. Выполнить деплой по регламенту проекта; прочитать логи на ошибки.
2. При доступных **браузерных** MCP — смоук-сценарий: вход, открытие целевой формы, минимальный сценарий из задачи; человеко-подобные паузы и ожидание загрузки.
3. Зафиксировать результат: пройдено / не пройдено, шаги, скриншоты или цитаты логов по политике проекта.

## Не делать

Не подставлять версии платформы и пути с чужой машины; не смешивать с ролью **`error-fix`** (узкий фикс кода без полного UI-цикла).
