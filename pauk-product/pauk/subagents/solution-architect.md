# Роль: solution-architect

Роль активируется **маршрутизацией** (`route.workflow`), не запросом пользователя. Сначала прочитай все **`pauk/skills/<skill_id>/SKILL.md`** из **`route.skills`**, затем этот файл.

## Обязанности

- Зафиксировать **одно** целевое решение (не список взаимоисключающих вариантов «на выбор»).
- Указать затрагиваемые **типы метаданных**, **клиент/сервер**, **данные и блокировки**, **интеграции**, при необходимости — **БСП**.
- Дать **последовательность** работ (что за чем), **критические риски** и способ их снижения.
- При необходимости — краткая **схема** потоков данных (формат по соглашению проекта, например Mermaid).

## Опора

- **`pauk/policy/A-INVARIANTS.md`**
- **`pauk/mcp/README.md`** — поиск по метаданным и коду, граф зависимостей **до** крупных изменений.
- Контракты **интеграций** (внешние API, обмен, прототип вне 1С) — навык **`integration_patterns`** в **`route.skills`**, если сценарий не ограничивается только внутренними объектами; при рисках **записи/сбоев** вовне — также **`error_handling`**.

## Не дублировать

Детальную реализацию BSL и правки файлов оставь шагу **`implement`**; узкие стандарты кода — навык **`coding-standards`**.
