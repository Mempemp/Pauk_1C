# Pauk_1C — фабрика оснастки ИИ для 1С

Репозиторий — **фабрика**: здесь проектируется сеть оснастки (навыки, маршрутизация, субагенты, MCP, политики). **Готовый продукт** для встраивания в рабочие репозитории лежит отдельно в **`pauk-product/`**: туда **не** входят `docs/`; в прикладном репо после установки не будет фабричной документации.

Цель продукта: качество и предсказуемость при **независимости от модели** и провайдера.

## Два контура

| Контур | Путь | Назначение |
|--------|------|------------|
| **Фабрика** | `docs/`, обсуждения, ADR | Источник правды для разработки оснастки |
| **Продукт** | `pauk-product/` | Копируемый набор: `.cursor/rules`, `pauk/routing`, `pauk/policy`, навыки, субагенты, MCP |

Правило Cursor для **этого** репозитория: `.cursor/rules/pauk-factory.mdc`.  
Правило для **целевого** репо после копирования: `pauk-product/.cursor/rules/pauk-entry.mdc`.

## Принципы (черновик)

1. **Модель — сменный адаптер**, не ядро: один контракт задач, разные исполнители.
2. **Навыки** — узкие пакеты знаний; в продукте — под `pauk/skills/`.
3. **Маршрутизация** — отдельная задача; сжатый каталог в `pauk-product/pauk/routing/SKILLS-CATALOG.md`.
4. **Субагенты + workflow** — см. `docs/SUBAGENTS.md` и перенос в `pauk-product/pauk/subagents/`.
5. **MCP** — опционально, слой над ядром; заготовка `pauk-product/pauk/mcp/`.
